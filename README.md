### 2. 🎯 청년 고립·은둔 위험군 예측 및 의사결정 지원 웹 서비스

**"데이터 기반으로 사회 문제를 조기에 발굴하고 정책 현장에 즉시 활용하는 ML 프로젝트"**

| 항목 | 내용 |
|------|------|
| **개발 기간** | 2026.01 (약 1주일)|
| **역할** | 풀스택 (데이터 분석, ML 모델링, 웹 서비스 구현) |
| **데이터** | 서울시 청년 5,513명 (원본 21개 변수 → 44개 특성) |
| **기술 스택** | Python, scikit-learn, Streamlit, joblib |
| **핵심 기능** | 개인 위험도 진단, 핵심 요인 분석, 모델 비교, 웹 대시보드 |



**🎯 핵심 성과**
- 🤖 **최고 성능 모델**: Random Forest (ROC-AUC 0.873, F1 0.523)
- 🔍 **고위험군 탐지율**: 59.8% (재현율 기준) - 실제 위험군 6명 중 3.6명 정확히 진단
- 💡 **주요 인사이트**: 교류 빈도, 경제 수준, 노동 여부 3가지 요인이 전체 위험의 42% 설명
- ⚡ **실무 적용**: Streamlit 대시보드로 정책 담당자·상담사 누구나 1초 안에 위험도 진단



**💡 주요 기술 경험**
- **불균형 데이터 처리 전문성**: class_weight 조정, F1-Score 최적화로 소수 클래스 정확 탐지
- **특성 엔지니어링**: 명목형/서수형 변수 분류 및 원-핫 인코딩으로 순서 왜곡 방지
- **모델 비교 분석**: 해석성(Logistic Regression) vs 성능(Random Forest) 트레이드오프 관리
- **하이퍼파라미터 튜닝**: GridSearchCV + 5-Fold 교차검증으로 n_estimators 150, max_depth 20 최적화
- **웹 서비스 구현**: Streamlit으로 슬라이더/드롭다운 기반 입출력 UI, joblib 모델 캐싱



**📚 배운 점** <br>
 ✅ 높은 정확도보다 **도메인 요구사항에 맞는 지표 선택** (Accuracy 74% → F1-Score 52%로 전환)<br>
 ✅ **해석 가능성의 가치**: 모델 성능만큼 "왜 이렇게 예측했는가"가 정책 현장에서 중요<br>
 ✅ 클래스 불균형 데이터에서 **Precision-Recall 균형**을 맞추는 실무 스킬<br>
 ✅ 기술 완성도만큼 **현장의 실제 요구사항을 이해하고 반영**하는 것의 중요성<br>

### 4. 🍔 LLM 기반 지능형 AI 키오스크 전환 시스템

**자연어·음성 기반 주문으로 기존 터치식 키오스크의 한계를 해결한 로컬 LLM 서비스 프로젝트**

| 항목 | 내용 |
|------|------|
| **개발 기간** | 2024.01 (약1주일) |
| **역할** | 풀스택 (LLM 설계, 음성 처리, 추천 로직, 웹 서비스 구현) |
| **데이터** | 자체 설계 메뉴 데이터 15종 (버거·사이드·음료, 영양·알레르기 포함) |
| **기술 스택** | Python, Streamlit, Ollama(Gemma2), Whisper, gTTS, Pydantic |
| **핵심 기능** | 음성·텍스트 주문, LLM 의도 분석, 맞춤 메뉴 추천, 음성 안내 |


**🎯 핵심 성과**
- 🤖 로컬 LLM 서비스 구현: Ollama(Gemma2) 기반 자연어 주문 분석 → 외부 API 없이 완전 로컬 환경 구축
- 🎙️ 다중 모달 인터페이스: 음성(STT)·텍스트 입력 모두 지원하는 키오스크 UX 설계
- 🍽️ 맞춤형 메뉴 추천: 예산·선호도·알레르기 조건을 반영한 메뉴 및 세트 자동 추천
- 🔊 음성 상호작용 완성: 추천 결과를 TTS로 안내하는 End-to-End 주문 흐름 구현
- 🛡️ 안정성 확보: 동시성 제어, 타임아웃, LLM 실패 시 폴백 로직으로 서비스 중단 방지


**💡 주요 기술 경험**
- **LLM 기반 의도 분석 설계**: 자연어 입력을 JSON 구조로 변환하여 예산·선호·알레르기 정보 추출
- **로컬 LLM 활용 전략**: Ollama 도입으로 비용·보안·프라이버시 문제 해결
- **음성 AI 파이프라인**: Whisper(STT) → LLM 분석 → gTTS(TTS) 흐름 통합
- **추천 시스템 로직 구현**: 필터링 + 우선순위 정렬 + 세트 조합 알고리즘 설계
- **동시성 제어**: 음성·텍스트 입력 중복 처리 방지를 위한 잠금 메커니즘 구현
- **웹 서비스 최적화**: Streamlit 캐싱(st.cache_data, st.cache_resource)으로 응답 속도 개선


**📚 배운 점** <br>
✅ 자연어 입력을 **정형 데이터(JSON)**로 변환하는 설계의 중요성<br>
✅ 음성·텍스트가 결합된 환경에서 **동시성·에러 처리의 필요성**<br>
✅ AI 기능 구현을 넘어 **실제 사용자를 고려한 UX 중심 설계 경험**<br>
✅ LLM·STT·TTS를 하나의 서비스로 통합하는 **End-to-End AI 시스템 구축 경험**<br>
